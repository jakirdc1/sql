CHAPTER FOUR

Demonstrate Understanding Of The Main Features Of Structured Programming

96. Stub

Stub is the portion of a decision table that lists all the conditions to be examined and the various possible actions to be taken.

97. Action Entry

Action entry is an indication of an action to be taken under a rule in a decision table.

98. Action Stub

Action stub is the list of actions in a decision table.

99. Condition Entry

Condition entry is an indication of the status of a relevant condition for a rule in a decision table.

100. Condition Stub

Condition stub is the list of conditions in a decision table.

101. Contradiction

Contradiction is a situation in a decision table in which the same combinations of conditions lead to different actions.

102. Decision Table

Decision table is a tool for planning and documenting processing that involves complex combinations of conditions. There are four components of decision table. They are: Action Entry, Action Stub, Condition Entry and Condition Stub.

103. Program Switch

Program switch is a field in memory, having the value of true or false that is used to record a condition.

104. Redundancy

Redundancy is a situation in which more than one rule of a decision table may be applied for a given combination of conditions. Redundancy exists when there are more rules than are necessary.

  1. Look for two rules that have the same actions
  2. Then if all condition entries but one are exactly the same, there is redundancy between the two rules

105. Rule

Rule is a part of decision table that indicates what actions are to be taken under a given combination of conditions.
