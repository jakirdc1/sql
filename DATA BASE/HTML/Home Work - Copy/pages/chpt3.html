Demonstrate Understanding Of The Main Features Of Structured Programming

Question: What is the comparison of value?
Answer: The comparison of two values is represented in a program flowchart by the decision outline. In structured programming, the condition in an IF-THEN-ELSE instruction usually consists of a comparison of two values; the condition in a DO WHILE instruction may also be such a comparison.

Question: What is combining conditions?
Answer: We are often confronted (challenged) with situations in which we must test more than one condition before we can select the correct course of action. In that situation we use different combinations of conditions to solve that confront which is called combining conditions.

Question: What is Pseudocode?
Answer: Pseudocode is an artificial and informal language that helps programmers develops algorithms. Pseudocode is a "text-based" detail (algorithmic) design tool for structured programming.

Question: How do we represent the logic for a simple sorting problem in pseudocode?
Answer: Suppose, X and Y are two variables where X may be larger than Y or Y may be larger than X. Now we have to write a pseudocode which can do sorting the two variables properly. 


		IF X < Y THEN
			move X to TEMP
			move Y to X
			move TEMP to Y
		ELSE
			(null)
		END IF
Question: Write down some looping structure.
Answer: 

while loop structure: 
		while(condition)
			{
			conditional code;
			}

for loop structure: 
		for(init; condition; update)
			{
			conditional code;
			}	
		[N.B. update means increment or decrement]
			
do while loop structure: 
		do
			{
			conditional code;
			}while(condition)

Question: What do you mean by sorting?
Answer:In computer science, arranging in an ordered sequence is called sorting. The most common uses of sorted sequences are:
  1. making lookup or search efficient
  2. making merging of sequences efficient
  3. enable processing of data in a defined order

Question: What is Boolean algebra and what is the use of it?
Answer:Boolean algebra a form of algebra in which symbols have the values of either true or false and are subject to the operations AND, OR, and NOT (also known as Boolean logic). In computer programming the Boolean algebra is used to define every event within two expressions.

Question: What do you mean by truth table?
Answer:The truth table is commonly used to represent the possible values of combinations of conditions. There are four rules which can make any kind of truth table.
  1. The expression A AND B is true if A is true and B is true.
  2. The expression A OR B is true if A is true or B is true or both are true.
  3. If A is true, then NOT A is false.
  4. If A is false, then NOT A is true.

Question: What do you mean by order of comparison or precedence?
Answer:A order of comparison is a type of sorting algorithm that only reads the list elements through a single abstract comparison operation (often a "less than or equal to" operator) that determines which of two elements should occur first in the final sorted list.

Question: What is input editing?
Answer:A programmer designing a program that will process data being entered into the computer for the first time must be aware of the necessity for editing the data. So input editing is a processing that checks the validity of data. There are some of the more widely used techniques for input editing:
  1. Sequence checking
  2. Restricted-value test
  3. Miscellaneous other tests

Question: What is sequence checking?
Answer:Sequence checking is a method for verifying that a file is in order by comparing key fields of consecutive records.

Question: What is restricted-value test?
Answer:Restricted-value test is a test applied to an input field to determine if its value is acceptable, as for example within a range of values.

Question: What is miscellaneous other tests?
Answer:Depending on the situation, any of a number of other tests may be performed on input data, which is called miscellaneous other tests.

Question: What do you mean by error routine?
Answer:Error routine is some instructions that are executed when an error is encountered during processing.

Question: What are the end-of-file checking techniques?
Answer:End-of-file is a name of condition when the data is finished to read. The computer will signal an end-of-file condition when it occurs. If this technique is not available or not appropriate in computer then the programmer must be prepared end-of-file checking techniques. Two other techniques can be used to detect the end-of-file condition. They are:
  1. Sentinel value and
  2. A counter

Question: What is sentinel value for the end-of-file checking techniques?
Answer:Sentinel value is a specific value placed in a field in a dummy record to signal the end-of-file.

Question: What is counter for the end-of-file checking techniques?
Answer:The program has to maintain a count of the number of records that are read and processed; when the count of the number of records processed equals the number expected, the processing of data records stops.
